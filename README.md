# Sistema de Gestão de Eventos

Projeto desenvolvido no curso de Análise e Desenvolvimento de Sistemas do SENAI como parte de uma proposta aplicada voltada à organização de eventos, com foco especial em cerimonialistas e no fluxo de trabalho de casamentos.

O objetivo do sistema é centralizar em um único lugar as informações que hoje costumam ficar espalhadas entre planilhas, mensagens, anotações e arquivos soltos. A proposta reúne cadastros de eventos, convidados, fornecedores, tarefas, contratos, controle financeiro e acompanhamento do andamento de cada cerimônia, buscando reduzir retrabalho e dar mais visibilidade ao processo.

## Sobre o projeto

O sistema nasce de um problema real de gestão: a dificuldade de acompanhar prazos, fornecedores, listas de convidados, confirmações de presença e dados financeiros sem perder controle das informações. A documentação do projeto foi construída para atender a esse contexto e orientar a evolução da solução ao longo das etapas do curso.

## O que o projeto busca resolver

- organização das informações do evento em um único ambiente;
- acompanhamento de tarefas e prazos;
- gestão de convidados e confirmação de presença (RSVP);
- controle de fornecedores e contratos;
- apoio ao planejamento financeiro do evento;
- visualização rápida do progresso geral da cerimônia.

## O que estamos desenvolvendo

Este projeto está sendo usado para consolidar conhecimentos de análise, modelagem e construção de software. Ao longo do trabalho, a equipe está praticando temas como levantamento de requisitos, modelagem entidade-relacionamento, definição de regras de negócio, organização de fluxos e tomada de decisões de escopo.

Também estamos aprendendo a transformar documentação em solução de software de forma gradual, validando o que faz sentido para a realidade do projeto antes de avançar para a implementação completa.

## Status atual

O projeto ainda está em fase de planejamento e documentação. Algumas partes já foram detalhadas nos artefatos, mas a implementação completa será construída por etapas. Neste momento, a stack final ainda não foi fechada, então o README evita assumir tecnologias que podem mudar durante a validação do projeto.
